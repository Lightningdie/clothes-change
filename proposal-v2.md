# 项目提案文档 v2.0 - 多界面功能扩展

## 项目概述

**项目名称**: 小猫猫的衣橱 (Clothes Change)

**项目类型**: React TypeScript Web 应用

**提案版本**: 2.0

**提案日期**: 2024

## 需求描述

### 核心需求

1. **主界面（首页）**
   - 作为应用的入口界面
   - 提供导航功能，跳转到各个功能模块
   - 展示应用的主要功能入口

2. **编辑穿搭界面（当前界面）**
   - 将现有的编辑界面保留为独立的编辑穿搭功能
   - 增加保存按钮，可以保存当前搭配
   - 支持保存多个穿搭方案

3. **查看保存穿搭界面**
   - 展示所有已保存的穿搭方案
   - 支持查看、编辑、删除已保存的穿搭
   - 可以预览穿搭效果

4. **个人信息界面**
   - 用户个人信息管理
   - 个人资料设置
   - 偏好设置

5. **上传服饰界面**
   - 支持上传自定义服饰图片
   - 管理已上传的服饰
   - 支持删除和编辑上传的服饰

## 功能详细设计

### 1. 主界面（HomePage）

**功能点：**
- 应用标题和欢迎信息
- 导航卡片/按钮：
  - 编辑穿搭（跳转到编辑界面）
  - 我的穿搭（跳转到查看保存穿搭界面）
  - 个人信息（跳转到个人信息界面）
  - 上传服饰（跳转到上传服饰界面）
- 美观的 UI 设计，使用卡片式布局

### 2. 编辑穿搭界面（EditOutfit）

**功能点：**
- 保留现有的所有编辑功能
- 新增保存按钮：
  - 保存当前搭配
  - 输入搭配名称
  - 保存搭配信息（包括选择的服饰类型）
- 显示当前搭配预览
- 支持撤销/重做操作

### 3. 查看保存穿搭界面（MyOutfits）

**功能点：**
- 展示所有已保存的穿搭列表
- 每个穿搭项显示：
  - 搭配名称
  - 创建时间
  - 预览图
  - 操作按钮（查看、编辑、删除）
- 支持搜索和筛选
- 支持编辑已保存的穿搭

### 4. 个人信息界面（Profile）

**功能点：**
- 用户基本信息展示
- 个人资料编辑
- 偏好设置
- 统计数据（保存的穿搭数量等）

### 5. 上传服饰界面（UploadClothes）

**功能点：**
- 图片上传功能
- 服饰信息填写：
  - 服饰名称
  - 服饰类型/分类
  - 描述
- 已上传服饰列表
- 支持删除和编辑

## 技术实现方案

### 路由管理

使用 React Router 进行路由管理：
- `/` - 主界面
- `/edit` - 编辑穿搭界面
- `/outfits` - 查看保存穿搭界面
- `/profile` - 个人信息界面
- `/upload` - 上传服饰界面

### 状态管理

- 使用 React Context API 或状态管理库管理全局状态
- 保存的穿搭数据存储在 localStorage 或 IndexedDB
- 上传的服饰信息存储在 localStorage

### 数据结构

**保存的穿搭数据结构：**
```typescript
interface SavedOutfit {
  id: string;
  name: string;
  clothTypes: ClothType[];
  createdAt: string;
  updatedAt: string;
  preview?: string;
}
```

**上传的服饰数据结构：**
```typescript
interface UploadedCloth {
  id: string;
  name: string;
  type: ClothType;
  imagePath: string;
  description: string;
  uploadedAt: string;
}
```
